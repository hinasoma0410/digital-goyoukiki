# デジタル御用聞き LP

岡山県内の外壁塗装会社・住宅リフォーム会社向け「元請け受注基盤構築サービス」の静的LPです。GitHub Pagesで公開できます。

## 1. ファイル構成

- `index.html`：LP本体
- `privacy.html`：プライバシーポリシー
- `404.html`：ページが見つからない場合の案内
- `assets/css/style.css`：色・文字・余白などの見た目
- `assets/js/main.js`：メニュー、フォーム、設定反映、アクセス解析
- `assets/images/`：写真や画像の保存場所
- `config/site-config.js`：運営者情報やURLなどの設定
- `robots.txt` / `sitemap.xml`：検索エンジン向け設定

## 2. ローカルで確認する方法

まず `index.html` をブラウザで開きます。フォーム送信など公開時に近い確認をする場合は、パソコンのターミナルでこのフォルダを開き、`python3 -m http.server 8000` を実行します。その後、ブラウザで `http://localhost:8000` を開きます。終了するときはターミナルで `Control + C` を押します。

## 3. 文章を変更する方法

主な文章は `index.html` にあります。変更前にファイルを複製するか、Gitで履歴を残してください。成果保証、未確認の実績・資格・顧客名は掲載しないでください。

## 4. 画像を変更する方法

1. 顔写真と見本サイト画像を `assets/images/` に保存します。
2. WebPまたはAVIF形式を推奨します。
3. `index.html` の `portrait-placeholder`、`profile-photo`、`work-placeholder` を `img` 要素へ置き換えます。
4. `width`、`height`、内容を説明する `alt` を付けます。ファーストビュー以外の画像には `loading="lazy"` を付けます。

実在企業の施工写真やAI生成写真を、確認済み実績として掲載しないでください。

## 5. LINE URLを設定する方法

`config/site-config.js` の `lineUrl` に、確認済みのLINE公式アカウントURLを入力します。空欄の間はボタンが無効になり、ダミーURLへ移動しません。

## 6. フォーム送信先を設定する方法

GitHub Pagesだけではフォームを受信できません。Formspreeなど外部フォームサービスを契約し、取得したHTTPSの送信先URLを `formEndpoint` に入力します。契約先の利用規約、保存場所、迷惑メール対策、送信テストを確認してください。秘密鍵やAPIキーは入力しないでください。

## 7. Google Analyticsを設定する方法

`gaMeasurementId` に `G-` から始まる測定IDを入力すると読み込まれます。空欄では読み込みません。利用開始時はCookieの扱いを確認し、必要に応じてプライバシーポリシーと同意方法を更新してください。

## 8. その他の設定

`config/site-config.js` でサイト名、運営者名、屋号、メール、電話、対応地域、料金、所在地、SNS URL、公開URLを変更できます。`publicUrl` は末尾の `/` なしでも構いません。

## 9. GitHubへアップロードする方法

1. GitHubで新しいリポジトリ（保管場所）を作ります。
2. このフォルダの中身をリポジトリ直下へ追加します。
3. 変更内容を確認し、コミット（変更履歴の保存）してGitHubへ送ります。

GitHub Desktopを使う場合は、このフォルダを追加し、変更の説明を入力して `Commit to main`、続いて `Publish repository` または `Push origin` を選びます。

## 10. GitHub Pagesを有効にする方法

1. GitHubの対象リポジトリで `Settings` → `Pages` を開きます。
2. `Build and deployment` の方法を `Deploy from a branch` にします。
3. 公開するブランチを `main`、フォルダを `/(root)` にして保存します。
4. 数分後に表示される公開URLを確認します。
5. 公開URLを `site-config.js`、`robots.txt`、`sitemap.xml` に反映します。

## 11. 独自ドメインの注意

GitHub Pagesの `Custom domain` にドメインを設定し、ドメイン管理会社側でもDNS設定を行います。HTTPSが有効になるまで待ち、旧URLからの重複やメール用DNS設定を壊さないよう、現在の管理者へ確認してください。

## 12. 公開前チェック

- 運営者名、屋号、所在地、メール、電話、公開日が正しい
- LINEボタンとフォームが実際に送信できる
- 外部フォームサービスの規約とプライバシー案内が合っている
- 顔写真、見本サイト画像、掲載許可がそろっている
- 実績、資格、経歴、顧客名に確認・掲載許可がある
- `https://TODO.example` がすべて公開URLに置き換わっている
- スマートフォンとパソコンで表示・操作を確認した
- Google Analytics利用時のCookie対応を確認した

## 13. 未設定のTODO一覧

- 運営者名、正式な屋号、公開可能な所在地
- 公開用メールアドレス、電話番号
- LINE公式アカウントURL、フォーム送信先
- GitHub Pagesまたは独自ドメインの公開URL
- Google Analytics測定ID、SNS URL（利用する場合のみ）
- 運営者の顔写真、経歴、資格、公開日
- 見本・実績の画像、説明、公開URL、掲載許可
- フォーム外部サービスの名称とプライバシー案内

`TODO` はフォルダ内検索でまとめて確認できます。

公開前の簡易チェックは、ターミナルで `node work/check-site.mjs` を実行すると再確認できます。

## 14. 更新前に履歴を残す方法

小さな変更でも、更新前にフォルダを複製して日付を付けるか、Gitでコミットしてください。問題が起きたときに以前の状態へ戻しやすくなります。個人情報や秘密鍵は履歴にも保存しないでください。
